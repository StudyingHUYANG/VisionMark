# 视觉候选切点实验

## 目标

为 VisionMark 的视频分析流程增加确定性的视觉候选切点检测，输出 `visualCuts[]`，给后续大模型分段提供边界参考。

当前实现位置：

- `server/services/visualCutDetector.js`
- `server/services/visual_cut_metrics.py`
- `server/services/videoAnalyzer.js`

## 输入格式

`getVisualCuts(frames)` 接收按时间排序的视频帧列表：

```js
[
  {
    framePath: "E:/a_work/VisionMark/downloads/BVxxx_visual_frames/visual_000001.jpg",
    time: 42.5
  }
]
```

字段说明：

- `framePath`: 本地帧图片路径。
- `time`: 当前帧对应的视频时间，单位秒。

## 输出格式

```js
[
  {
    time: 42.5,
    score: 0.76,
    reasons: ["visual_change", "ssim_drop", "visual_peak", "scene_change"],
    method: "ssim",
    metrics: {
      ssimDiff: 0.82,
      histDiff: 0.41,
      phashDiff: 0.34
    },
    previousTime: 41.5
  }
]
```

字段说明：

- `time`: 候选切点时间，取变化后第一帧时间。
- `score`: 综合视觉变化分数，范围 `0-1`。
- `reasons`: 触发原因。
- `method`: 主导指标，可能为 `ssim` / `histogram` / `phash`。
- `metrics`: 三类指标的原始变化量。
- `previousTime`: 上一帧时间，表示变化发生在 `previousTime` 到 `time` 之间。

## 使用算法

1. SSIM

   对相邻帧灰度图计算结构相似度，使用 `ssimDiff = 1 - ssim`。该指标对画面结构、构图、镜头切换敏感。

2. Histogram Diff

   对 RGB 三通道分别计算归一化直方图，使用 L1 距离得到 `histDiff`。该指标对色彩、亮度、背景大面积变化敏感。

3. pHash

   将帧缩放为 `32x32` 灰度图，做 DCT 后取低频区域生成感知哈希，使用 Hamming 距离得到 `phashDiff`。该指标能忽略部分噪声，捕获整体视觉变化。

4. Scene Change + 视觉峰值检测

   综合分数：

   ```text
   score = 0.45 * ssimDiff + 0.35 * histDiff + 0.20 * phashDiff
   ```

   阈值：

   ```text
   threshold = max(baseThreshold, mean(score) + 1.35 * std(score))
   ```

   再从分数序列中选择局部峰值，并用 `minGapSeconds` 合并过近候选点。

## 推荐阈值

当前默认值：

```js
{
  baseThreshold: 0.55,
  peakStdFactor: 1.35,
  maxDynamicThreshold: 0.92,
  ssimThreshold: 0.60,
  histThreshold: 0.38,
  phashThreshold: 0.32,
  minGapSeconds: 15.0,
  warmupSeconds: 1.5,
  ignoreEndSeconds: 15.0,
  ignoreEndMinDuration: 60.0
}
```

推荐配置：

- 普通 B 站视频：`1 fps` 抽帧，`baseThreshold = 0.55`，`minGapSeconds = 15`。
- 快剪、游戏、舞台视频：`2 fps` 抽帧，`baseThreshold = 0.50`，`minGapSeconds = 5-8`，否则会漏掉短镜头。
- 讲解、PPT、访谈视频：`1 fps` 抽帧，`baseThreshold = 0.60`，`minGapSeconds = 15-20`，减少字幕/翻页误判。
- 最终分段不要直接照搬 `visualCuts`，建议把它作为大模型分段提示和人工复核候选点。

## 容易误判的场景

- 弹幕、字幕、贴纸、进度条等覆盖层突然出现。
- 闪白、黑场、转场特效、片头片尾卡片。
- 游戏、体育、舞蹈等高速运动视频，镜头没切但像素变化很大。
- PPT 翻页或大字标题切换，视觉强变化但语义可能仍属于同一段。
- 同一机位访谈、讲解视频，语义换段但画面变化很小，容易漏检。
- B 站常见“一键三连”“关注”等固定插片容易被识别为强切点。

## 接入方式

`VideoAnalyzer.analyzeVideo()` 现在在关键帧抽取后执行：

1. `extractVisualProbeFrames(videoPath, bvid, duration)`
2. `analyzeVisualCuts(frames)`
3. 将 `visual_cuts` 和 `visual_cut_stats` 写入分析结果
4. 将候选切点追加到 Qwen VL 提示词中，作为分段边界参考

API 分析结果中新增：

```js
{
  visual_cuts: [],
  visual_cut_stats: {}
}
```
